## Context

Feverless 是一款 iOS SwiftUI + SwiftData 应用，用于追踪儿童体温和用药记录。目前数据完全封闭在设备内，无法导出备份或与医生共享。本设计需在零外部依赖的前提下实现 CSV 导入导出，并同步优化 ProfileView 的 UI。

## Goals / Non-Goals

**Goals:**
- 将指定孩子、指定时间范围的体温和用药记录导出为单一 CSV 文件
- 通过系统 ShareSheet 分发导出文件（保存、分享、AirDrop 均支持）
- 从 CSV 文件导入记录到当前选中孩子，自动检测日期格式，重复记录跳过
- 格式错误时提供行级错误提示
- 优化 ProfileView：孩子卡片展示最近体温，底部增加数据管理区

**Non-Goals:**
- 不支持 `.numbers` 或 `.xlsx` 原生格式（无系统 API）
- 不支持跨孩子批量导入导出
- 不支持导出时包含孩子基本信息（姓名、生日）
- 不支持云端自动备份

## Decisions

### 决策 1：CSV Schema——单文件 + 类型鉴别列

**选择**：单文件，第一列为 `record_type`（`temperature` 或 `medication`），其余列覆盖两种记录的所有字段，不适用的字段留空。

```
record_type,timestamp,value,method,medication_type,concurrent_temperature,notes
temperature,2026-05-20T10:30:00+08:00,38.5,ear,,,头疼
medication,2026-05-20T11:00:00+08:00,,,ibuprofen,38.5,
```

**放弃的替代方案**：
- 分段式（`## Temperature Records` 注释分隔）：解析复杂，Numbers/Excel 无法直接识别结构
- 两个独立文件：破坏"单一文件"约定，ShareSheet 分发两个文件体验差

**理由**：Numbers 和 Excel 均能直接打开，空列直观，解析逻辑最简单。

### 决策 2：时间戳格式

**导出**：统一使用 ISO 8601 含时区偏移（`yyyy-MM-dd'T'HH:mm:ssXXXXX`），确保跨地区无歧义。

**导入自动检测顺序**：
1. ISO 8601 含时区（`2026-05-20T10:30:00+08:00`）
2. ISO 8601 UTC（`2026-05-20T10:30:00Z`）
3. ISO 8601 无时区（`2026-05-20T10:30:00`）→ 补充本地时区
4. 中文常用（`2026/05/20 10:30`）→ 补充本地时区
5. 纯日期（`2026-05-20`）→ 时间补 `00:00:00` + 本地时区

无法匹配任何格式则报告行号和原始值。

**理由**：优先精确格式，降级处理手动编辑场景，只在完全无法解析时报错。

### 决策 3：重复检测键

**选择**：`(timestamp, record_type)` 作为复合去重键，精确到秒。

**理由**：同一孩子在同一秒不可能有两条同类型记录。用 UUID 去重会因重新导出生成新 UUID 而失效。

### 决策 4：导入入口——ProfileView 数据管理区

**选择**：ProfileView 底部新增 `Section("数据管理")`，标题动态显示 `"<孩子名> 的数据"`，包含"导出数据"和"导入数据"两行。操作绑定到当前选中孩子。

**放弃的替代方案**：
- 孩子行滑动操作：滑动操作已有"编辑"和"删除"，再加两项过于拥挤
- 独立标签页：功能量不足以独占一个 Tab

### 决策 5：导出分发——ShareSheet

**选择**：将 CSV 写入临时目录（`FileManager.default.temporaryDirectory`），通过 `UIActivityViewController` 呈现系统 ShareSheet。

**理由**：iOS 原生方案，支持保存到文件 App、AirDrop、邮件、信息等，无需额外权限。

### 决策 6：导入预览——先解析后确认

**选择**：解析完成后展示预览 Sheet（将导入 X 条体温、Y 条用药，跳过 Z 条重复），用户确认后才写入 SwiftData。

**理由**：给用户掌控感，避免意外覆盖数据。错误在解析阶段提前拦截，预览阶段只展示合法数据。

## Risks / Trade-offs

- **大文件内存**：全量导出/导入时若记录数极多（>10000 条）可能有内存压力 → 当前场景（儿童发烧记录）数据量极小，不做流式处理
- **时区歧义**：导入无时区的 CSV 时补充本地时区，跨时区使用可能有细微偏差 → 已在检测顺序中优先匹配含时区格式
- **CSV 特殊字符**：`notes` 字段可能含逗号或换行 → 导出时对含特殊字符的字段加双引号，遵循 RFC 4180
